# 管家记忆 Schema 技术文档 — 长尾偏好谱需求规格

> **定位**：这不是 persona 文件。这是 `管家记忆.md` 的数据结构规范 + 对应的 Skill 注入需求。

## 1. 偏好维度 (Preference Dimensions)

以下四个维度的数据由管家在交互中语义提取后写入 `管家记忆.md`，随后所有 Skill 调用前读取出对应的过滤参数。

### 1.1 口味忌口 (Taste)

| 字段 | 类型 | 示例 | 注入 Skill |
|---|---|---|---|
| `taste_tolerance` | enum: `重辣/中辣/微辣/无辣/甜` | `无辣` | `generic_poi_searcher` → 过滤辣味品类 |
| `dietary_restrictions` | `string[]` | `["香菜", "海鲜过敏"]` | 结果二次过滤 |
| `cuisine_preference` | `string[]` | `["日料", "火锅", "轻食"]` | 品类优先排序 |

### 1.2 通勤习惯 (Commute)

| 字段 | 类型 | 示例 | 注入 Skill |
|---|---|---|---|
| `walking_tolerance_meters` | `int` | `800` | `destination_anti_pitfall` → 超阈值时 `transport="打车"` |
| `transport_priority` | enum: `步行优先/打车优先/地铁优先` | `打车优先` | 同上，设 `transport` 默认值 |

### 1.3 消费预算 (Budget)

| 字段 | 类型 | 示例 | 注入 Skill |
|---|---|---|---|
| `price_level` | enum: `经济/中端/高端` | `中端` | POI 结果二次过滤（按 `signature_dishes[0].price` 近似判定） |
| `rating_cutoff` | `float` (0-5) | `4.3` | `generic_poi_searcher` 的 `min_rating` 参数 |

### 1.4 健康作息 (Lifestyle)

| 字段 | 类型 | 示例 | 注入 Skill |
|---|---|---|---|
| `hydration_interval_minutes` | `int` | `90` | `time_master` schedule 中插入 WATER 提醒节点 |
| `medication_schedule` | `{time: "HH:MM", name: string}[]` | `[{"time":"08:00","name":"降压药"}]` | `task_reminder_skill` 的 MED 事件注册 |

## 2. 存储格式要求

`管家记忆.md` 采用 Markdown 表格格式（人类可读 + LLM 可解析）：

```markdown
## 口味
| 字段 | 值 |
|---|---|
| taste_tolerance | 无辣 |
| dietary_restrictions | 香菜, 海鲜过敏 |
| cuisine_preference | 日料, 轻食 |

## 通勤
| 字段 | 值 |
|---|---|
| walking_tolerance_meters | 800 |
| transport_priority | 打车优先 |

## 预算
| 字段 | 值 |
|---|---|
| price_level | 中端 |
| rating_cutoff | 4.3 |
```
