## 🔧 Skill 能帮我扩展能力吗？

**Skill 的本质**：Skill 是预设的 prompt 模板 + 工作流指引，它不能凭空增加新工具。但它能：

| Skill能做到的 | Skill做不到的 |
|-------------|-------------|
| 让我更系统地执行复杂多步骤任务 | 给我摄像头/麦克风访问权限 |
| 预置API调用模板和参数规范 | 绕过API Key认证 |
| 引导我按最佳实践写代码 | 编译原生iOS/Android应用 |
| 串联多个Agent并行工作 | 访问硬件传感器 |
| 规范化输出格式（如MCP协议） | 替代企业商业合作谈判 |

### 对49项功能的影响

- **✅ 可直接做的13项**：Skill 有帮助 — 预设一个 `implement-feature` skill，标准化"理解需求→设计方案→写代码→写测试"流程
- **🔑 需要Key的16项**：Skill 有帮助 — 预设 `amap-integration` skill，把高德API端点/参数/错误处理模板化
- **❌ 做不到的9项**：Skill **无法改变** — 没有硬件就是没有硬件

### 建议创建的 Skill

| Skill 名称 | 解决的问题 |
|-----------|----------|
| `amap-poi` | 高德POI搜索标准化接入，预置所有POI类目编码、参数模板 |
| `amap-route` | 高德路径规划标准化接入，驾车/公交/步行/骑行 |
| `amap-weather` | 高德天气API标准化接入 |
| `itinerary-generator` | 自然语言→多日行程的LLM prompt模板 |
| `anti-hallucination` | 双引擎防幻觉：LLM输出→API校验→过滤的标准化流程 |
| `feature-implement` | 从待办清单选取功能→设计方案→写代码→测试的标准化流程 |
