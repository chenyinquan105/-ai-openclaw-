## 第2页：为什么做 + 做了什么（~40秒）⭐ 总

**大标题**: 一个懂你、守护你的AI管家

**左右对比布局**:

| 😞 痛点 | ✅ 我们的方案 |
|---|---|
| 本地生活决策碎片化：选餐厅、排时间、查路线分散在多个App | **一站式闭环**：搜索 → 排程 → 路径 → 提醒，全程AI接管 |
| 每次都不记得你的偏好：口味、忌口、预算反复设置 | **长期记忆驱动**：四维度偏好谱（口味/通勤/预算/健康）自动读写 |
| 突发异常全靠手忙脚乱：暴雨改计划、堵车重排、排队过长白等 | **自动容灾Plan B**：异常实时感知 → 替换/跳过/推迟方案一键确认 |
| 健康提醒形同虚设：喝水吃药没人督促，错过无人知 | **五步强闭环**：提醒→确认→再催→45分钟无响应→紧急联络人 |

**口播要点**: "本地生活有四个核心痛点——碎片化、没记忆、没预案、没督促。我们用一个具备长期记忆和自主容灾能力的AI管家，一次性解决这四个问题。"

---

## 第3页：四大核心功能（~60秒）⭐ 分

**大标题**: 四大能力，覆盖全场景

**四宫格卡片布局**:

### 🍜 智能商户搜索
- 多维约束检索：品类、评分、距离、价格、忌口
- 偏好自动注入：偏好菜系置顶、忌口隐藏、超预算降权
- 支持店名模糊纠偏 + 高德真实POI数据源

### 🗓️ 出行排程规划
- 多任务并发时间线求解（如"理发→咖啡→宠物洗澡"一键排程）
- 空间重心策略优化访问顺序，最小化折返
- 含：出行指南（步行超3000m建议打车）+ 防坑校验 + 多日景点排程

### 💊 健康监督提醒
- 喝水/吃药定时推送（间隔可配置）
- 五步强闭环：到期提醒 → 确认准备 → 确认服用 → 延时再催 → 升级紧急联络人
- SSE实时推送 + 守护线程独立轮询

### 🛡️ 异常容灾Plan B
- 实时监测：暴雨、堵车、停电、排号异常
- 三种自动策略：SWAP（替换商户）/ BYPASS（跳过节点）/ POSTPONE（推迟时间）
- 弹窗确认 → 一键执行

**口播要点**: "四大能力分为两组：上面两个帮你'找对地方、走对路线'，下面两个帮你'守住健康、应对意外'。每个功能都在Web控制台可实操演示，待会儿大家会看到。"

---

## 第4页：技术亮点（~40秒）⭐ 分

**大标题**: 两个核心技术支柱

### 🧠 记忆驱动 — 怎么「懂你」
- **四维度偏好谱**：口味（辣度/忌口/偏好菜系）、通勤（步行容忍度/交通优先级）、预算（价格档位/评分底线）、健康（喝水间隔/吃药时间）
- **存储方式**：Markdown表格文件，人类可读 + LLM可解析，零数据库依赖
- **自动闭环**：交互前读取注入 → 交互后LLM语义检测偏好变化 → 自动更新
- **景点模板学习**：用户手动调整排程后自动保存为模板，下次匹配直接复用

### 🏗️ 架构设计 — 怎么「可靠」
- **三层解耦**：OpenClaw TypeScript网关（控制面）+ Python Flask后端（9个Skill模块）+ HTML5沙盒控制台（表现面）
- **时空沙盒**：虚拟时钟支持拖拽快进、倍速、跳转，沙盒仿真与静态排程双轨并行
- **Demo零依赖**：所有商户/天气/排队数据本地Mock，确保演示稳定可复现

**口播要点**: "技术上就两件事：一是记忆系统让AI越用越懂你，二是解耦架构让系统稳定可靠。下面直接看演示。"
